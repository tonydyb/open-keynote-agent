# Design: Rename Python Package and CLI Command

## Overview

This is a mechanical rename. No business logic changes. The steps are:

1. Rename the source directory.
2. Update all import statements.
3. Update `pyproject.toml`.
4. Update documentation and spec files.
5. Reinstall the package to regenerate entry points.
6. Verify.

## File Changes

### Source directory

```text
src/open_mac_agent/  →  src/open_keynote_agent/
```

All files inside the directory move with it. The `.egg-info` directory under `src/` should be deleted; it will be regenerated by `uv sync`.

### pyproject.toml

| Field | Old value | New value |
|---|---|---|
| `project.name` | `open-mac-agent` | `open-keynote-agent` |
| `project.description` | references macOS file organization | references Keynote automation |
| `project.authors` | `Open Mac Agent` | `Open Keynote Agent` |
| `project.keywords` | include `macOS`, `file organizer` | include `Keynote`, `macOS` |
| `project.scripts` key | `oma` | `oka` |
| `project.scripts` value | `open_mac_agent.cli:app` | `open_keynote_agent.cli:app` |

### Source files — import updates

Every `from open_mac_agent.` or `import open_mac_agent.` in source files becomes `from open_keynote_agent.` or `import open_keynote_agent.`.

### cli.py — string literals

The `version` command prints `open-mac-agent 0.1.0` — update to `open-keynote-agent 0.1.0`.

The `hello` command refers to `open-mac-agent` and `oma` — update both.

The `create_run_session` calls pass `"oma organize"` and `"oma ask"` as command strings — update to `"oka organize"` and `"oka ask"`.

### Test files — import updates

All `from open_mac_agent.` and `import open_mac_agent.` become `from open_keynote_agent.` and `import open_keynote_agent.`.

### egg-info cleanup

Delete `src/open_mac_agent.egg-info/` before reinstalling. Running `uv sync` regenerates it as `src/open_keynote_agent.egg-info/`.

## Verification

```bash
uv sync
uv run oka --help
uv run oka organize <tmp-folder> --dry-run
uv run pytest
uv run ruff check .
```

All five must pass. The `oma` command must no longer exist in the virtual environment after reinstall.
